Can we use Machine Learning Classifications to Reliably Classify Spotify Songs by Type of Popularity?¶

Sidney Howard-Clinton

Spotify is perhaps the most widely recognized streaming service available. Its wide catalogue of songs and ease of accessibility make it the first choice for music enthusiast. One of the more innovative features of Spotify is its abilities to allow users to create and share their own public playlists with the community. However, Spotify additionally creates and curates their own playlists to share with the community. Two of these playlists are Spotifys Viral Top 50 and Top 200. As you might have guessed the Top 200 playlist is an amalgamation of the top 200 most streamed songs at any given moment. The Viral Top 50 however is a bit more ambiguous. According to a Spotify employees response in a post on the Spotify forums, the Viral 50 chart has been created to measure the success of music in terms of user sharing combining analytics on blogs, social media etc.

Additionally, Spotify analyzes songs to discern their features and using their own libraries, Spotify allows developers to access these songs features using their "Spotipy" python library. These features include acousticness, danceability, tempo, energy etc. More will be discussed about these features later. For this reason, I thought it would be fascinating to use these audio features Spotify has collected and use machine learning classification algorithms to attempt to classify songs as belonging to either the Viral 50 or Top 200 playlist. Is it even possible to discern a difference between viral songs and top songs? What classification algorithm would be optimal? Our human intuition would have us believe that the most viral songs would be indistinguishable from the top songs. If our classification is at all successful, it begs the question to what extent can a songs virality and popularity be quantified based on its auditory characteristics.

Data Collection and Processing¶

The playlist data we'll be using for this analysis was collected by user Dhruvil Dave on Kaggle. Spotify publishes the Top 200 and Viral 50 charts every 2-3 days. This user has created a CSV file with the entire collection of charts dating back from January 1, 2017. The CSV contains around 26 million entries and has the songs name, rank, its playlist (either Viral 50 or Top 200) and its respective URL (which we will need to access the audio features) along with a few other data points which wont be needed for this particular analysis.

Our dataframe has a column named trend that stores a songs position movement within the playlist. Songs can either move up, down or be newly entered into the playlist. For this analysis, I will only use newly entered songs to remove the possibility of a few biggest hits having a disproportionate impact on our data.

The following block is where we will get the auditory attributes of all the songs in our dataframe. Using the songs URL, we make a web request to get songs attributes from Spotify and then concatenate those columns to our main dataframe. For 100,000 songs this process took around 2 and a half hours. Because of this I chose to export the data frame to CSV so that I could rerun my code without having to repeating scrape all this data. If a new set of data is desired, the code can simply be rerun but be warned it will take lots of time to collect the attributes.

Danceability is defined by Spotify as: how suitable a track is for dancing based on a combination of musical elements including tempo, rhythm stability, beat strength, and overall regularity. A value of 0.0 is least danceable and 1.0 is most danceable.

The two sets have a very similar average danceability value.

Energy is defined by Spotify as: a measure from 0.0 to 1.0 and represents a perceptual measure of intensity and activity. Typically, energetic tracks feel fast, loud, and noisy. For example, death metal has high energy, while a Bach prelude scores low on the scale. Perceptual features contributing to this attribute include dynamic range, perceived loudness, timbre, onset rate, and general entropy.

The two sets have a very similar average energy value.

Key is defined by Spotify as: The key the track is in. Integers map to pitches using standard Pitch Class notation. E.g. 0 = C, 1 = C/D, 2 = D, and so on. If no key was detected, the value is -1.

The average key is the same between both sets.

Loudness is defined by Spotify as: The overall loudness of a track in decibels (dB). Loudness values are averaged across the entire track and are useful for comparing relative loudness of tracks. Loudness is the quality of a sound that is the primary psychological correlate of physical strength (amplitude). Values typically range between -60 and 0 db.

The two sets have a similar average loudness value.

Mode is defined by Spotify as: the modality (major or minor) of a track, the type of scale from which its melodic content is derived. Major is represented by 1 and minor is 0.

The average mode is the same between both sets.

Speechiness is defined by Spotify as: the presence of spoken words in a track. The more exclusively speech-like the recording (e.g. talk show, audio book, poetry), the closer to 1.0 the attribute value. Values above 0.66 describe tracks that are probably made entirely of spoken words. Values between 0.33 and 0.66 describe tracks that may contain both music and speech, either in sections or layered, including such cases as rap music. Values below 0.33 most likely represent music and other non-speech-like tracks.

The two sets have a similar average speechiness value.

Acousticness is defined by Spotify as: A confidence measure from 0.0 to 1.0 of whether the track is acoustic. 1.0 represents high confidence the track is acoustic.

The two sets have a similar average acousticness value.

Instrumentalness is defined as Spotify as: whether a track contains no vocals. "Ooh" and "aah" sounds are treated as instrumental in this context. Rap or spoken word tracks are clearly "vocal". The closer the instrumentalness value is to 1.0, the greater likelihood the track contains no vocal content. Values above 0.5 are intended to represent instrumental tracks, but confidence is higher as the value approaches 1.0.

It seems the viral 50 songs are on average 3 times more instrumental than top 200 songs! This may be a key feature in classify the songs later.

Liveness is defined by Spotify as: the presence of an audience in the recording. Higher liveness values represent an increased probability that the track was performed live. A value above 0.8 provides strong likelihood that the track is live.

The two sets have a similar average liveness value.

Valence is defined by Spotify as: A measure from 0.0 to 1.0 describing the musical positiveness conveyed by a track. Tracks with high valence sound more positive (e.g. happy, cheerful, euphoric), while tracks with low valence sound more negative (e.g. sad, depressed, angry).

The two sets have a similar average valence value.

Tempo is defined by Spotify as: The overall estimated tempo of a track in beats per minute (BPM). In musical terminology, tempo is the speed or pace of a given piece and derives directly from the average beat duration.

The two sets have a similar average tempo value.

Finally, time Signature is defined by Spotify as An estimated time signature. The time signature (meter) is a notational convention to specify how many beats are in each bar (or measure). The time signature ranges from 3 to 7 indicating time signatures of "3/4", to "7/4".

The average time signature of both sets is a standard 4/4 time signature.

Upon observation, there seems to be a predictable extreme similarity between the attributes of viral 50 songs and top 200 songs. The only standout feature being instrumentalness. The viral 50 songs are on average 3 times more instrumental than top 200 songs. Perhaps this will be the key factor in our machine learning models in determining classification.

A decision tree is a supervised learning algorithm that specializes in classification problems. It is similar to how a flow chart works where data points are separated into two smaller categories at a time. Consequently, I expect this to be great classification algorithm for my problem. From our output, we can observe the model has an accuracy of 0.67 overall and an f1-score of 0.69 for the top 200 set and 0.66 for the viral 50 set. F1-score is calculating using the precision and recall scores of the model. Precision is calculated using the formula: precision = true positive / (true positive + false positive). Recall is calculated using the formula: recall = true positive/ (true positive + false negative). A f1-score of 1 indicates a perfect model. Accuracy tells us the overall performance of the model. For each classification algorithm I generate a confusion matrix to clearly visualize the performance of the algorithm.

It seems decision trees performed fairly well with an accuracy of 0.67. This means that the decision tree was right in classifying songs 67% of the time.

Conclusion and Further Exploration¶

From the preliminary analysis of song attributes between the top 200 and viral 50, it seemed that the differences were so negligible between the playlists that classification would be futile. However, machine learning has proved itself yet again by demonstrating the ability to classify songs, based on their auditory attributes, with almost 70% accuracy. Random forest classification being the standout algorithm. Its accuracy would have surely increased if provided with more data. Unfortunately, due to resource constraints, scraping data was an extremely time expensive task. I would like to see this experiment repeated with much larger datasets. For now, I believe this is a fascinating insight into machine learning. It would seem intuitive to us that viral songs would be essentially the same as the most popular songs. However, machine learning has proven that songs do have auditory attributes that contribute to their popularity and virality. Perhaps a songs instrumentalness is such an attribute with our preliminary graphs showing it to be the main difference between a viral song and a popular one. If the differences between instrumentalness were minimized, would machine learning algorithms still be effective for this problem?
